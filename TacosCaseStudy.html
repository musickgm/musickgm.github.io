Case Study

Taco's Tacos

Project Overview


The Goal:

Design a menu/ordering app that prioritizes users who are in a hurry and may be interested in making quick repeat orders.

My Role:

Lead UX Designer for the hypothetical restaurant Taco's Tacos from conception to delivery.

  • Responsibilities

    • Conducting interviews
    • Paper and digital wireframing
    • Low and high-fidelity prototyping
    • Conducting usability studies
    • Accounting for accessibility
    • Iterating on designs


    • Understanding the User

    Persona:

    Michael is a young professional who needs to quickly and efficiently order and pickup because of their schedule constraints.

    User Journey:

    Mapping Michael's user journey revealed how helpful it would be for users to be able to see predictions for order time as well as notifications for the order timeline.

  • User Research - Pain Points:

    • Inaccurate Availability - When users attempt to order something from a restaurant later to find that it is out of stock or unavailable, it causes frustration.
    • Inaccurate Wait Time - Users often make plans around when their order will be ready or decide to order from somewhere based on wait times. When these aren't accurate, it creates a user pain point.
    • Need for Repeat Orders - Navigating through the app for large orders can take time. This can be greatly reduced for families or groups that consistently make the same order.
    • Poor App Navigation - Users become frustrated when navigating is not intuitive or when the layout does not make sense.


    • Starting the Design

    Paper Wireframes:

    In this step, taking the time to draft different iterations of the screen of the app on paper ensured that the elements that made it to digital wireframes would be helpful in addressing user pain points. For the favorites screen, I prioritized a quick and easy ordering process to help users save time.

    Digital Wireframes:

    As the initial design phase continued, I made sure to base screen designs on feedback and findings from the user research.

    Quickly viewing and navigating through favorite orders was a key user need to address in the designs.

    Low-Fidelity Prototype:

    The low-fidelity prototype connected the primary user flow of placing an order from a saved "favorites" order so that the prototype could be used in a usability study with users.


  • Usability Study - Findings:

    Round 1:

    1. (1) Users need consolidated navigation for their favorites/previous orders.
    2. (2) Users need better cues for when and how they can edit an order.
    3. (3) Users need better cues for how they can save an edited order.


    Round 2:

    1. (1) Users need to be able to add new items to a saved order.
    2. (2) Users need to see the order total on each screen.
    3. (3) Users need to see the cost of each item based on count on each screen.

    • Refining the Design

    Mockups:

    An "Edit Order" page was added after the first usability study. After the second usability study, I added a feature for users to add new items to their order on the edit screen.

    The second usability study revealed a frustration with not being able to see the order total on each screen. To address this, the order total has been added to the "Saved Orders", "Edit Order", and "Order Details" screens.

  • Accessibility Considerations:

    1. (1) Colors for buttons, backgrounds, and text have been adjusted to promote accessibility.
    2. (2) Animations between frames have been set to 300ms. This amount of time is in line with recommendations from the World Wide Web Consortium.
    3. (3) Layouts have been designed to be consistent. All call to action buttons have the same size, shape, and color to promote accessibility.
